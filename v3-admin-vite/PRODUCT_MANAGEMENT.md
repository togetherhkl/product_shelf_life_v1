# 产品管理页面功能说明

## 页面功能概述

产品管理页面是一个完整的CRUD（增删改查）系统，用于管理产品信息。基于后端的Product模型和相关API接口实现。

## 主要功能

### 1. 查询功能

- **多条件搜索**：
  - 产品名称（模糊搜索）
  - 扫码值（条形码或二维码，模糊搜索）
  - 分类选择（下拉框，精确匹配）
  - 状态筛选（正常、即将过期、已过期）

- **特殊查询**：
  - 即将过期产品列表（7天内到期）
  - 已过期产品列表

### 2. 数据展示

- **表格列**：
  - 产品名称
  - 扫码值（以标签形式显示）
  - 分类（关联显示分类名称）
  - 批次号
  - 生产日期
  - 到期日期（带状态色彩标识）
  - 产品图片（缩略图，支持预览）
  - 创建者（管理员用户名）
  - 创建时间

- **状态标识**：
  - 绿色：正常产品
  - 黄色：即将过期（7天内）
  - 红色：已过期

### 3. 新增/编辑功能

- **表单字段**：
  - 产品名称（必填）
  - 扫码值（必填，唯一性）
  - 分类选择（必填，下拉框）
  - 批次号（可选）
  - 生产日期（可选，日期选择器）
  - 到期日期（可选，日期选择器）
  - 产品图片URL（可选）
  - 产品描述（可选，多行文本）

### 4. 删除功能

- 支持单条记录删除
- 删除前确认提示

### 5. 分页功能

- 支持分页显示
- 可调整每页显示数量
- 页码跳转

## 技术特性

### 1. 图片处理

- 自动拼接baseURL处理相对路径图片
- 支持http/https绝对路径图片
- 图片预览功能

### 2. 日期处理

- 自动计算过期状态
- 日期格式化显示
- 过期提醒功能

### 3. 关联数据

- 自动加载分类列表供选择
- 显示创建者信息
- 分类名称显示

### 4. 状态管理

- 加载状态提示
- 操作成功/失败提示
- 表单验证

## API集成

### 使用的API接口：

1. `getProductListApi` - 获取产品列表（支持分页和搜索）
2. `createProductApi` - 新增产品
3. `updateProductApi` - 更新产品
4. `deleteProductApi` - 删除产品
5. `getExpiredProductsApi` - 获取已过期产品
6. `getExpiringSoonProductsApi` - 获取即将过期产品
7. `getCategoryListApi` - 获取分类列表

### 数据结构：

- 响应格式：`{ products: ProductData[], total: number }`
- 支持分页参数：`skip`, `limit`
- 支持搜索参数：`name`, `barcode_or_qr`, `category_id`

## 用户交互

### 1. 搜索操作

- 输入搜索条件后点击"查询"按钮
- "重置"按钮清空搜索条件
- "即将过期"和"已过期"快捷按钮

### 2. 数据管理

- "新增产品"按钮打开新增对话框
- 表格中的"修改"按钮编辑记录
- 表格中的"删除"按钮删除记录

### 3. 图片操作

- 鼠标悬停显示图片预览
- 点击图片可放大查看

## 注意事项

1. **后端依赖**：需要后端API返回正确的数据格式（包含total字段的分页数据）
2. **分类依赖**：需要分类管理功能正常工作
3. **权限控制**：某些操作需要管理员权限
4. **图片处理**：需要正确配置VITE_BASE_URL环境变量
5. **日期格式**：使用YYYY-MM-DD格式
